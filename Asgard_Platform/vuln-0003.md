# IDOR Vulnerability in Dashboard Widget Management

**ID:** vuln-0003
**Severity:** MEDIUM
**Found:** 2026-03-25 05:56:34 UTC
**Target:** /workspace/Platform/Modules/Dashboard
**Endpoint:** /dashboard/grid
**Method:** GET, POST
**CWE:** CWE-639
**CVSS:** 5.4

## Description

The Dashboard module's widget management system contains an Insecure Direct Object Reference vulnerability. The EloquentWidgetRepository's findForUser method retrieves widgets based on user_id without proper authorization checks. While the middleware checks permissions for dashboard access, the actual widget retrieval relies solely on user_id matching, allowing authenticated users to potentially access other users' dashboard configurations if they can manipulate the user_id parameter.

## Impact

An authenticated user could potentially access and modify other users' dashboard widget configurations, leading to unauthorized data access and potential privilege escalation if dashboard widgets contain sensitive information or administrative functionality.

## Technical Analysis

The vulnerability exists in the widget management system where user-specific widget data is stored and retrieved based on user_id. The EloquentWidgetRepository::findForUser method uses a simple WHERE clause to find widgets by user_id without verifying that the requesting user has permission to access that specific user's data. While the routes are protected by middleware checking 'dashboard.index' permission, this only verifies general dashboard access, not specific user data authorization.

## Proof of Concept

1. Authenticate as a regular user
2. Access the dashboard endpoint which loads user-specific widgets
3. Intercept the request and modify the user_id parameter sent to widget retrieval
4. Observe that widgets for other users can be accessed if the user_id is manipulated

```
import requests

# This is a conceptual PoC - actual implementation would depend on API structure
def test_widget_idor(base_url, session_cookie, target_user_id):
    headers = {'Cookie': f'session={session_cookie}'}
    
    # Attempt to access another user's widget configuration
    response = requests.get(f'{base_url}/api/dashboard/widgets?user_id={target_user_id}', headers=headers)
    
    if response.status_code == 200:
        print(f"IDOR vulnerability confirmed - accessed user {target_user_id}'s widgets")
        return True
    return False
```

## Code Analysis

**Location 1:** `Modules/Dashboard/Repositories/Eloquent/EloquentWidgetRepository.php` (lines 15-18)
  Widget retrieval without authorization check
  ```
      public function findForUser($userId)
    {
        return $this->model->whereUserId($userId)->first();
    }
  ```

  **Suggested Fix:**
```diff
-     public function findForUser($userId)
-     {
-         return $this->model->whereUserId($userId)->first();
-     }
+     public function findForUser($userId)
+     {
+         // Verify the authenticated user has permission to access this user's data
+         $authenticatedUserId = auth()->id();
+         if ($userId != $authenticatedUserId && !auth()->user()->hasRole('admin')) {
+             abort(403, 'Unauthorized access to user widget data');
+         }
+         
+         return $this->model->whereUserId($userId)->first();
+     }
```

**Location 2:** `Modules/Dashboard/Http/Controllers/Admin/DashboardController.php` (line 49)
  Proper usage with authenticated user ID
  ```
          $widget = $this->widget->findForUser($this->auth->id());
  ```

## Remediation

1. Implement proper authorization checks in EloquentWidgetRepository::findForUser to verify the requesting user has permission to access the target user's data
2. Use the authenticated user's ID from the session rather than accepting user_id as a parameter
3. Add role-based checks to ensure users can only access their own widget data unless they have admin privileges
4. Implement input validation to prevent user_id manipulation
