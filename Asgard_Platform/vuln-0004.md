# Path Traversal Vulnerability in AsgardCMS Media Module File Path Handling

**ID:** vuln-0004
**Severity:** HIGH
**Found:** 2026-03-25 05:56:47 UTC
**Target:** /workspace/Platform/Modules/Media
**Endpoint:** Various Media module endpoints using file paths
**Method:** GET, POST
**CWE:** CWE-22
**CVSS:** 8.8

## Description

The AsgardCMS Media module contains a path traversal vulnerability in the file path handling logic that allows attackers to read and write files outside the intended directory structure. The vulnerability exists in the Imagy.php class's getFilenameFor method which constructs file paths from user-controlled input without proper sanitization.

The method processes MediaPath objects containing user-controlled file paths and constructs new filenames for thumbnail generation. By manipulating the input path, an attacker can traverse directories and access sensitive files or overwrite critical system files.

## Impact

Successful exploitation allows attackers to read sensitive files (configuration files, source code, credentials) and potentially write files to arbitrary locations on the server. This can lead to information disclosure, privilege escalation, remote code execution, and complete system compromise.

## Technical Analysis

The vulnerability exists in the Imagy.php getFilenameFor method which processes MediaPath objects:

```php
private function getFilenameFor(MediaPath $path, $thumbnail)
{
    $filenameWithoutPrefix = $this->removeConfigPrefix($path->getRelativeUrl());
    $filename = substr(strrchr($filenameWithoutPrefix, '/'), 1);
    $folders = str_replace($filename, '', $filenameWithoutPrefix);
    
    return config('asgard.media.config.files-path') . $folders . $this->newFilename($path, $thumbnail);
}
```

The method extracts folder paths from user-controlled input and concatenates them with the configured files-path. An attacker can supply a path containing directory traversal sequences (../) to escape the intended directory. The vulnerability is triggered when processing file paths for thumbnail generation and file operations.

## Proof of Concept

To reproduce the vulnerability:

1. Identify endpoints that process file paths (thumbnail generation, file retrieval)
2. Craft a malicious file path containing traversal sequences: ../../../etc/passwd
3. Submit the path to vulnerable endpoints that use the getFilenameFor method
4. The method will construct a path outside the intended directory
5. Access sensitive files or overwrite system files

The vulnerability can be exploited through:
- Thumbnail generation endpoints that process user-controlled paths
- File retrieval endpoints using MediaPath objects
- Any functionality that passes user input to the Imagy class

## Remediation

1. Implement path sanitization in getFilenameFor method:
   - Use realpath() or equivalent to resolve paths
   - Validate that final path remains within intended directory

2. Add path validation before file operations:
   - Check that resolved paths are within allowed directories
   - Reject paths containing traversal sequences

3. Use Laravel's built-in path security features:
   - Utilize Storage facade with secure path handling
   - Implement proper access controls for file operations

4. Add input validation for file paths:
   - Validate path format before processing
   - Reject paths with suspicious patterns
