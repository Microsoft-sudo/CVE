# Cross-Site Scripting (XSS) via Unsafe v-html Usage in Media Module Move Dialog

**ID:** vuln-0005
**Severity:** HIGH
**Found:** 2026-03-29 18:28:09 UTC
**Target:** /workspace/Platform
**Endpoint:** /api/media/folders/all-nestable
**Method:** GET
**CWE:** CWE-79
**CVSS:** 7.3

## Description

A critical Cross-Site Scripting (XSS) vulnerability was identified in the Media module's MoveMediaDialog.vue component. The vulnerability exists due to improper use of the v-html directive to render user-controlled folder names without proper sanitization.

The vulnerability chain flows as follows:
1. User-controlled folder names are stored in the database
2. The AllNestableFolderController.php endpoint retrieves folder names via the listsFlattened() method
3. The MoveMediaDialog.vue component fetches these folder names via API call
4. The folder names are rendered unsafely using v-html directive

The listsFlattened() method in NestedFoldersCollection.php processes folder names without HTML encoding, allowing malicious payloads to be stored and retrieved. When rendered via v-html, any HTML/JavaScript content in folder names executes in the victim's browser context.

This vulnerability allows attackers to execute arbitrary JavaScript code in the context of authenticated users, potentially leading to session hijacking, account takeover, and unauthorized administrative actions.

## Impact

Successful exploitation allows attackers to:
- Execute arbitrary JavaScript in the context of authenticated users
- Steal session cookies and authentication tokens
- Perform actions on behalf of the victim user
- Potentially gain administrative access if the victim has elevated privileges
- Conduct phishing attacks by modifying the application's UI
- Access and exfiltrate sensitive user data

The impact is particularly severe for administrative users who have access to sensitive functionality within the Media module and other administrative interfaces.

## Technical Analysis

The vulnerability exists in the data flow from folder creation to rendering:

1. **Data Source**: Folder names are user-controlled inputs stored in the database
2. **API Endpoint**: `/api/media/folders/all-nestable` returns folder data via AllNestableFolderController.php
3. **Data Processing**: NestedFoldersCollection.php processes folder names without HTML encoding
4. **Vulnerable Sink**: MoveMediaDialog.vue renders folder names using v-html directive

The v-html directive in Vue.js renders raw HTML without sanitization, making it vulnerable to XSS when used with untrusted data. The folder names retrieved from the API are treated as trusted content when they are actually user-controlled.

The vulnerability is compounded by the fact that folder names are typically considered benign, but attackers can create folders with malicious names containing XSS payloads.

1. Create a folder with an XSS payload as the name:
   - Navigate to the Media module file manager
   - Create a new folder with name: `<img src=x onerror=alert(document.domain)>`
   - Alternatively, use a more sophisticated payload for demonstration

2. Trigger the vulnerable component:
   - Select any media file in the file manager
   - Click the "Move" action to open the MoveMediaDialog

## Code Analysis

**Location 1:** `Modules/Media/Assets/js/components/MoveMediaDialog.vue` (line 21)
  Unsafe v-html usage rendering user-controlled folder names
  ```
                              <span v-html="item"></span>
  ```

  **Suggested Fix:**
```diff
-                             <span v-html="item"></span>
+                             <span v-text="item"></span>
```

**Location 2:** `Modules/Media/Http/Controllers/Api/AllNestableFolderController.php` (line 23)
  Source of user-controlled folder data
  ```
          $folders = $this->folder->allNested()->nest()->listsFlattened('filename', null, 0, $array, '--- ');
  ```

**Location 3:** `Modules/Media/Support/Collection/NestedFoldersCollection.php` (line 90)
  Folder name processing without HTML encoding
  ```
              $item_string = str_repeat($indentChars, $level) . $item->$column;
  ```

## Remediation

1. **Immediate Fix**: Replace v-html with v-text or proper sanitization
   - In MoveMediaDialog.vue, change `<span v-html="item"></span>` to `<span v-text="item"></span>`
   - This will render the content as plain text instead of HTML

2. **Input Validation**: Implement server-side validation for folder names
   - Add HTML tag filtering in folder creation/update endpoints
   - Reject folder names containing HTML tags or JavaScript

3. **Output Encoding**: Implement proper output encoding in the API
   - Encode folder names before returning them in API responses
   - Use HTML entity encoding for special characters

4. **Content Security Policy**: Implement CSP headers to mitigate XSS impact
   - Restrict script execution to trusted sources only
   - Use nonce-based CSP for inline scripts

5. **Security Review**: Conduct comprehensive security review of all v-html usage
   - Identify and fix similar vulnerabilities across the application
   - Implement safe rendering patterns for dynamic content

The safest approach is to avoid v-html entirely when rendering user-controlled data and use proper text rendering with v-text or mustache syntax.
