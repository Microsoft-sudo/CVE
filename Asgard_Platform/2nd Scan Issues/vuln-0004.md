# Hard-coded Credentials in User Factory

**ID:** vuln-0004
**Severity:** HIGH
**Found:** 2026-03-29 08:52:04 UTC
**Target:** /workspace/Platform
**Endpoint:** N/A - Development configuration issue
**Method:** N/A
**CWE:** CWE-798
**CVSS:** 7.3

## Description

A security issue was identified where hard-coded bcrypt password hashes are used in the UserFactory for testing purposes. While this is intended for development environments, the presence of hard-coded credentials poses a security risk if the code is deployed to production without proper configuration.

## Impact

If deployed to production without modification, this could allow:
- Unauthorized access using known credentials
- Bypass of authentication mechanisms
- Potential account takeover if the hard-coded password is known

## Technical Analysis

The vulnerability exists in `database/factories/UserFactory.php` where a hard-coded bcrypt hash is used:

```php
'password' => '$2y$10$TKh8H1.PfQx37YgCzwiKb.KjNyWgaHb9cbcoQgdIVFlYg7B77UdFm', // secret
```

This hash corresponds to the plaintext password "secret". While factory files are typically used only in development and testing environments, their presence in production codebases can pose security risks.

## Proof of Concept

This is not an exploitable vulnerability in the traditional sense but represents a security anti-pattern. In development environments, test accounts created with this factory would have the password "secret".

```
# This is a security anti-pattern, not an exploitable vulnerability
# The issue is the presence of hard-coded credentials in the codebase

def demonstrate_risk():
    print("Hard-coded credentials found in UserFactory")
    print("Password hash: $2y$10$TKh8H1.PfQx37YgCzwiKb.KjNyWgaHb9cbcoQgdIVFlYg7B77UdFm")
    print("Plaintext equivalent: 'secret'")
    print("Risk: Medium - depends on deployment context")
```

## Remediation

1. Remove hard-coded credentials from factory files
2. Use environment variables or configuration files for test credentials
3. Implement proper credential management for different environments
4. Use random password generation in factories:

```php
'password' => bcrypt(Str::random(10)),
```

5. Ensure factory files are excluded from production deployments
6. Implement environment-specific configuration for test data
