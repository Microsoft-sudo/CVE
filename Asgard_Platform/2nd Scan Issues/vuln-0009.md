# Business Logic Vulnerability - User-Controlled Sorting and Ordering

**ID:** vuln-0009
**Severity:** MEDIUM
**Found:** 2026-03-29 09:28:21 UTC
**Target:** /workspace/Platform
**Endpoint:** Various API endpoints
**Method:** GET
**CWE:** CWE-20
**CVSS:** 5.0

## Description

The application allows users to control sorting parameters without proper validation, which could potentially lead to business logic vulnerabilities including resource exhaustion or unexpected behavior.

## Impact

User-controlled sorting parameters could allow:
- Resource exhaustion through complex sorting operations
- Potential timing attacks through crafted sorting criteria
- Unexpected application behavior through invalid sorting parameters
- Potential information disclosure through error messages

## Technical Analysis

Multiple repositories accept user-controlled sorting parameters without proper validation:

```php
if ($request->get('order_by') !== null && $request->get('order') !== 'null') {
    $order = $request->get('order') === 'ascending' ? 'asc' : 'desc';
    $roles->orderBy($request->get('order_by'), $order);
}
```

While this is standard functionality, it lacks:
- Validation of allowed sorting columns
- Rate limiting on sorting operations
- Protection against resource-intensive sorting operations

## Proof of Concept

An attacker could potentially exploit sorting functionality by:
1. Requesting complex sorting operations on large datasets
2. Using non-indexed columns to cause performance issues
3. Attempting to sort by columns that trigger errors

## Remediation

1. Implement allowed sorting column validation
2. Add rate limiting for sorting operations
3. Validate sorting direction parameters
4. Implement query complexity limits

Example fix:
```php
$allowedColumns = ['id', 'email', 'created_at', 'updated_at'];
$allowedDirections = ['asc', 'desc'];

if (in_array($request->get('order_by'), $allowedColumns) && 
    in_array($request->get('order'), $allowedDirections)) {
    $roles->orderBy($request->get('order_by'), $request->get('order'));
}
```
