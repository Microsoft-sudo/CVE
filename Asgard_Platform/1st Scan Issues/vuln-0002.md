# DOM-based Cross-Site Scripting (XSS) in Media Module Dropzone

**ID:** vuln-0002
**Severity:** LOW
**Found:** 2026-04-04 18:41:22 UTC
**Target:** /workspace/Platform
**Endpoint:** Modules/Media/Assets/js/dropzone.js
**Method:** N/A (Client-side vulnerability)
**CWE:** CWE-79
**CVSS:** 3.7

## Description

A DOM-based Cross-Site Scripting (XSS) vulnerability was identified in the Dropzone.js file used by the Media module. The vulnerability occurs when user-controlled file size information is directly inserted into DOM elements using innerHTML without proper sanitization.

The vulnerability is located at line 433 where file.size (which could potentially be manipulated or contain malicious content in certain edge cases) is directly assigned to node.innerHTML. Additionally, the createElement call at line 436 uses string concatenation that could potentially be exploited if user input influences the template strings.

## Impact

An attacker could potentially execute arbitrary JavaScript code in the context of the user's browser if they can manipulate file metadata or influence the template strings. This could lead to:
- Session hijacking by stealing authentication tokens
- Defacement of the application interface
- Redirecting users to malicious websites
- Performing actions on behalf of the user

## Technical Analysis

The vulnerability exists in two locations:

1. **Line 433**: Direct assignment to innerHTML without sanitization:
```javascript
node.innerHTML = this.filesize(file.size);
```

2. **Line 436**: String concatenation in createElement call:
```javascript
file._removeLink = Dropzone.createElement("<a class=\"dz-remove\" href=\"javascript:undefined;\" data-dz-remove>" + this.options.dictRemoveFile + "</a>");
```

While file.size is typically numerical, in certain edge cases or if the browser/file system allows manipulation of file metadata, this could be exploited. The second vulnerability vector involves template strings that could be influenced through configuration options.

## Code Analysis

**Location 1:** `Modules/Media/Assets/js/dropzone.js` (line 433)
  Unsanitized innerHTML assignment with potentially user-controlled data
  ```
                                  node.innerHTML = this.filesize(file.size);
  ```

  **Suggested Fix:**
```diff
-                                 node.innerHTML = this.filesize(file.size);
+                                 node.textContent = this.filesize(file.size);
```

**Location 2:** `Modules/Media/Assets/js/dropzone.js` (line 436)
  String concatenation in DOM creation with potential XSS risk
  ```
                                  file._removeLink = Dropzone.createElement("<a class=\"dz-remove\" href=\"javascript:undefined;\" data-dz-remove>" + this.options.dictRemoveFile + "</a>");
  ```

  **Suggested Fix:**
```diff
-                                 file._removeLink = Dropzone.createElement("<a class=\"dz-remove\" href=\"javascript:undefined;\" data-dz-remove>" + this.options.dictRemoveFile + "</a>");
+                                 file._removeLink = Dropzone.createElement('a');
+                                 file._removeLink.className = 'dz-remove';
+                                 file._removeLink.href = 'javascript:undefined;';
+                                 file._removeLink.setAttribute('data-dz-remove', '');
+                                 file._removeLink.textContent = this.options.dictRemoveFile;
```

## Remediation

1. Replace innerHTML with textContent for numerical values
2. Implement proper output encoding for all dynamic content
3. Use DOM manipulation methods instead of string concatenation
4. Validate and sanitize all configuration options
5. Implement Content Security Policy (CSP) headers

Example fixes:

```javascript
// Fix for line 433 - use textContent instead of innerHTML
node.textContent = this.filesize(file.size);

// Fix for line 436 - use proper DOM methods
file._removeLink = Dropzone.createElement('a');
file._removeLink.className = 'dz-remove';
file._removeLink.href = 'javascript:undefined;';
file._removeLink.setAttribute('data-dz-remove', '');
file._removeLink.textContent = this.options.dictRemoveFile;
```
