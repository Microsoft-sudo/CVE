# Authentication Bypass and Account Enumeration Vulnerabilities in Sentinel Auth System

**ID:** vuln-0010
**Severity:** CRITICAL
**Found:** 2026-03-25 06:12:04 UTC
**Target:** /workspace/Platform/Modules/User
**Endpoint:** /auth/login, /auth/activate/{userId}/{code}, /auth/reset/{id}/{code}
**Method:** GET, POST
**CWE:** CWE-203
**CVSS:** 10.0

## Description

Critical authentication vulnerabilities were identified in the AsgardCMS authentication system using Cartalyst/Sentinel. The vulnerabilities include account enumeration through error message differentiation, activation code brute force attacks due to lack of rate limiting, and password reset IDOR vulnerabilities.

The vulnerabilities were discovered through static code analysis of the AuthController and authentication flow implementation. The system exposes information about account existence and status through differential error responses, and lacks proper rate limiting on sensitive authentication endpoints.

## Impact

Successful exploitation allows attackers to enumerate valid user accounts, brute force activation codes to activate malicious accounts, and potentially manipulate password reset flows to compromise user accounts. These vulnerabilities collectively enable account takeover and unauthorized access to the system.

## Technical Analysis

The vulnerabilities exist in multiple components of the authentication system:

1. Account Enumeration: Different error messages for invalid email vs. valid but incorrect password reveal account existence
2. Activation Brute Force: The /auth/activate/{userId}/{code} endpoint lacks rate limiting, allowing brute force attacks
3. Password Reset IDOR: User ID parameter in reset flow can be manipulated to target other users
4. Session Management: Potential session fixation risks in Sentinel's session handling

The authentication system relies on Cartalyst/Sentinel which has different security patterns than Laravel's built-in authentication, creating unique vulnerability surfaces.

## Proof of Concept

To reproduce these vulnerabilities:

1. Account Enumeration:
   - Send login request with invalid email: "Account not found" error
   - Send login request with valid email but wrong password: "Invalid credentials" error
   - The different messages reveal valid accounts

2. Activation Brute Force:
   - Obtain a valid user ID through enumeration or other means
   - Brute force activation codes by sending requests to /auth/activate/{userId}/{code}
   - No rate limiting allows rapid code guessing

3. Password Reset IDOR:
   - Start password reset for own account but capture the reset link
   - Modify the user ID parameter in the reset URL to target another user
   - Complete the reset flow to change another user's password

## Code Analysis

**Location 1:** `Modules/User/Http/Controllers/AuthController.php` (lines 8-24)
  Authentication endpoints without rate limiting
  ```
  $router->group(['prefix' => 'auth'], function (Router $router) {
    # Login
    $router->get('login', ['middleware' => 'auth.guest', 'as' => 'login', 'uses' => 'AuthController@getLogin']);
    $router->post('login', ['as' => 'login.post', 'uses' => 'AuthController@postLogin']);
    # Register
    if (config('asgard.user.config.allow_user_registration', true)) {
        $router->get('register', ['middleware' => 'auth.guest', 'as' => 'register', 'uses' => 'AuthController@getRegister']);
        $router->post('register', ['as' => 'register.post', 'uses' => 'AuthController@postRegister']);
    }
    # Account Activation
    $router->get('activate/{userId}/{activationCode}', 'AuthController@getActivate');
    # Reset password
    $router->get('reset', ['as' => 'reset', 'uses' => 'AuthController@getReset']);
    $router->post('reset', ['as' => 'reset.post', 'uses' => 'AuthController@postReset']);
    $router->get('reset/{id}/{code}', ['as' => 'reset.complete', 'uses' => 'AuthController@getResetComplete']);
    $router->post('reset/{id}/{code}', ['as' => 'reset.complete.post', 'uses' => 'AuthController@postResetComplete']);
    # Logout
    $router->get('logout', ['as' => 'logout', 'uses' => 'AuthController@getLogout']);
});
  ```

## Remediation

1. Implement rate limiting on all authentication endpoints, especially activation and password reset
2. Standardize authentication error messages to prevent account enumeration
3. Add ownership validation in password reset flow to prevent IDOR
4. Implement strong activation code generation with sufficient entropy
5. Add session fixation protection mechanisms
6. Conduct security review of Cartalyst/Sentinel configuration
